Model Architecture Optimization

To maximize representation capacity within the parameter constraints, several transformer improvements were implemented:

  • 7-Layer Architecture: Increasing network depth (L=7, d_model=128) improves hierarchical representation compared to the shallow baseline.
  • Weight Tying: Embedding weights are tied directly with the output language modeling head. The redundant weight matrix is stripped from the saved checkpoint to prevent double-counting of parameters, and restored transparently on loading.
  • Rotary Positional Embeddings (RoPE): Parameter-free relative positional encodings replace absolute position tables, saving 16,384 parameters and improving context length generalization.
  • RMSNorm: Replaces LayerNorm to speed up computation and save parameter bias terms.
  • SwiGLU Activation: Llama-style gated SiLU activations in the MLP block with d_ff = 8/3 * d_model to increase model capacity.
  • Bias-Free Layers: Biases in all linear projection layers are removed.

Tokenizer & Data Compression

Raw byte tokenization splits Hindi characters into 3 separate tokens, exhausting the 128-token context window rapidly. We trained a custom 4,096-vocabulary Byte-Pair Encoding (BPE) tokenizer on the training corpus:

  • Compression Efficiency: Compresses 159,225 bytes of text to 65,953 tokens, achieving a **2.41x compression ratio** (bytes per token).
  • Context Expansion: A block size of 128 tokens covers approximately **309 bytes of text** (a 2.4x expansion over the baseline context window).
  • Lossless Guarantee: A byte-level fallback handles unseen UTF-8 character bytes dynamically, ensuring the model never encounters out-of-vocabulary crashes on test data.
  • Encoding Cache: Storing tokenized word chunks in an O(1) cache speeds up corpus preprocessing from minutes to under 0.5 seconds.

Training Recipe & Hyperparameters

The model is trained with an optimized configuration designed to maximize data consumption and convergence speed within 2,000 steps:

  • AdamW Optimizer: Weight decay (0.1) is applied only to 2D weight matrices (excluding 1D norm parameters and embedding matrices).
  • Cosine Decay with Warmup: The learning rate ramps up linearly for 100 steps from 0 to a peak value of 1.5e-3, then decays in a cosine curve to a minimum value of 1.5e-4 by step 2,000.
  • Gradient Clipping: Gradients are clipped at a max norm of 1.0 to prevent optimization instability.
  • Batch Size: The batch size is increased to 32, allowing the model to train on 8.19 million tokens (over 2.8 epochs) in under 15 minutes of CPU execution.

Parameter Deduplication Implementation

To ensure strict parameter limit compliance when counting directly from the state dict keys of the checkpoint file, the redundant tied weight matrix is stripped during serialization and restored on load:

# Overridden in model.py (GPT class)
def load_state_dict(self, state_dict, strict=True):
    if self.cfg.tie_weights and "head.weight" not in state_dict:
        state_dict = state_dict.copy()
        state_dict["head.weight"] = state_dict["tok_emb.weight"]
    return super().load_state_dict(state_dict, strict=strict)

# Implemented in train.py (Checkpoint saving)
state_dict = model.state_dict()
if cfg.tie_weights:
    state_dict = {k: v for k, v in state_dict.items() if k != "head.weight"}
torch.save({"model": state_dict, "config": ...}, args.out)
